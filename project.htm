Upcoming Project


Overview

My project idea is to design a tool that can read AFL player statistics from a database and present them in an easy to read format to help Fantasy football players. The stats would include disposals, goals and fantasy points for each game and these will be presented in a line graph. The second display will be a boxplot which shows quartiles. The reason for the box plot is that currently the best available gauge is a player’s average which is not always an accurate representation. Users will also be able to filter the data they are looking at by venue and opponent to make the data and analysis more specific.

Motivation

There are 128,725 teams registered in AFL Fantasy in 2021. This number does not include players of other formats such as SuperCoach and Draftstars. These players are fanatical and despite this there are no good tools for researching and making decisions when it comes to creating a team and making trades. There is one which is $20 per year and all it does is show averages and give breakevens which are readily available on the internet. Given the fanatical nature of these players and the winning prize being a car I think that a much better tool could be made and it would be popular with players.

Description

AFL Fantasy is a numbers game. You have a set amount of money to make a team at the start of the season and players score points based on their stats. Player prices fluctuate week to week based on their performance as well so a strategy that is required to win is to pick players when they are expected to rise in price and trade players out when they are about to drop in price. There are head to head leagues which people run with their friends for cash and the person that has the highest overall score for the year will win a car.

The primary purpose of this app is to have a simple visual aid to help make the decisions required. The addition of filters to view more specific data can really allow someone to get a leg up on their opponent if they do get it right. I envision this as a database which is updated weekly with the player statistics and also feeds into a UI which displays the data as easy to read graphs. Ideally, there would also be commentary displayed based on the stats such as “Player X has never kicked less than 3 goals when playing against Team Y” or “Player X usually struggles against team Y with only 1 score over 90 in 7 games”. Further to this, and this is an option which presents the biggest challenge, we could have a bot that makes suggestions on which players are due to have some good or bad form. I will dot point and give a brief description of the possible features below.

Features

Score and stats analysis – As described above. Visual aids to help select players for your team.
Team Sheets – Primarily to be used in the pre season. This tool will allow players to pick teams and play around with them before the season starts without having to change their actual team.
News updates – Fantasy relevant news updates from AFL clubs such as injuries, potential debuts, role changes and training reports.
Best 22 on form – A team that updates each week with the players in each position with the highest scores in the last 3 rounds. A quick reference guide.
Bunnies – A section which details the players that always have great games against certain teams.
Kryptonite – A section which details the players that always perform poorly against certain teams.
Captain suggestions – Captains earn double points so a section that outlines the players best placed to get huge scores each week would be required reading for anyone that wants to really win a car.

Tools & Tech

Microsoft Excel can store and present the data the way that I want. I need to determine if there is a tool that can read the data and present it in a user friendly UI with appropriate menus and filtering options or if it is possible to code something like this. Whether there are any security issues with storing the data in Excel is another question. If Excel is not a viable option this idea could require building our own database and UI from scratch.

Skills Required

    Coding due to having to build the tool from the ground up.
    Analytics to provide the commentary around the insights that I would like to be shown.
    Automation so that we can have the data update itself every week rather than manually entering the stats of 400 players every Sunday night so that the app is fully functional all week for users
    Security as players will probably have to enter some kind of details to be able to use the tool. This data needs to be protected for privacy reasons.
    Mobile app development and web development so that we can have a web based version and a phone version.
    Testing so that we can identify bugs before releasing to the public.

I think that most of these aren’t too much of an issue to learn with materials that are available online. Security is something that I wouldn’t want to take risks with so professional advice or services may be required.

Outcome

The finished product would be a subscription service or free service with ads. I think because there is nothing like this on the market at the moment it will be very popular very quickly. I would set the release date to be around January in whichever year it is ready so that there is plenty of time for word to spread before the AFL season starts. I can also see this being used by punters so the market may actually be larger than initially intended.
